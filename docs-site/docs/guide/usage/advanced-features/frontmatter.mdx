# frontmatter

HarnSpec 在每个Spec顶部使用 YAML frontmatter作为结构化元数据。

## 什么是frontmatter？

frontmatter是 YAML 格式的机器可读元数据：

```markdown
---
status: in-progress
created: 2025-11-02
tags: [api, security]
priority: high
---

# 您的Spec标题

[Spec内容...]
```

## 核心字段

### status（必需）

Spec的当前状态。

**有效值：**
- `planned` - 尚未开始
- `in-progress` - 当前正在处理
- `complete` - 工作已完成
- `archived` - 工作已归档

**示例：**
```yaml
status: in-progress
```

### created（必需）

Spec创建的日期（ISO 8601 格式）。

**格式：** `YYYY-MM-DD`

**示例：**
```yaml
created: 2025-11-02
```

当您使用 `harnspec create` 时，这会自动填充。

## 可选字段

### tags

用于分类和过滤的标签数组。

**示例：**
```yaml
tags: [api, security, mvp]
```

**用法：**
```bash
harnspec list --tag=api
```

### priority

工作的重要性级别。

**有效值：**
- `low`
- `medium`
- `high`
- `critical`

**示例：**
```yaml
priority: high
```

**用法：**
```bash
harnspec list --priority=high
```

### completed

工作完成的日期（自动添加）。

**格式：** `YYYY-MM-DD`

**示例：**
```yaml
completed: 2025-11-05
```

当您设置 `status: complete` 时，这会自动添加。

## 自定义字段

您可以在 `.harnspec/config.json` 中定义自定义字段：

```json
{
  "frontmatter": {
    "custom": {
      "assignee": "string",
      "reviewer": "string",
      "epic": "string",
      "sprint": "number",
      "estimate": "string"
    }
  }
}
```

**支持的类型：**
- `string`
- `number`
- `boolean`
- `array`

**用法：**
```bash
harnspec create feature --field assignee=alice --field sprint=42
harnspec list --field epic=PROJ-123
```

详情请参见[自定义字段](/docs/guide/usage/advanced-features/custom-fields)。

## 视觉徽章

HarnSpec 从frontmatter自动生成视觉徽章：

```markdown
> **Status**: ⏳ 进行中 · **Priority**: 高 · **Created**: 2025-11-02 · **Tags**: api, security
```

**状态图标：**
- 📅 计划中
- ⏳ 进行中
- ✅ 已完成
- 📦 已归档

当您使用 `harnspec update` 时，这些徽章会自动更新。

## 完整示例

```yaml
---
status: in-progress
created: 2025-11-02
completed: 
tags: [api, authentication, security]
priority: high
assignee: alice
reviewer: bob
epic: PROJ-123
sprint: 42
---

# 用户认证 API

> **Status**: ⏳ 进行中 · **Priority**: 高 · **Created**: 2025-11-02  
> **Tags**: api, authentication, security · **Assignee**: alice · **Reviewer**: bob

[Spec内容...]
```

## 编辑frontmatter

### 通过 CLI

推荐的方法：

```bash
harnspec update 001 --status=complete --priority=critical
```

这会更新frontmatter和视觉徽章。

### 手动编辑

您也可以直接在文件中编辑frontmatter：

1. 在编辑器中打开 `README.md`
2. 修改 `---` 标记之间的 YAML
3. 保存文件

注意：手动编辑不会自动更新视觉徽章。运行 `harnspec update` 进行同步。

## 最佳实践

:::tip 从最小开始
从 `status` 和 `created` 开始。仅在需要时添加字段。
:::

:::tip 一致地使用标签
为您的团队建立标签约定。示例：`api`、`ui`、`backend`、`security`。
:::

:::tip 定期更新状态
保持状态最新。使用 `harnspec list` 查看进行中的内容。
:::

:::tip 不要过度自定义
谨慎添加自定义字段。每个字段都是要维护的开销。
:::
