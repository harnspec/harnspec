# 查找Spec

学习如何使用搜索、列表命令和 MCP 服务器发现和导航Spec以供 AI 代理访问。

## 快速发现

### 列出所有Spec

```bash
# 显示所有Spec
harnspec list
```

输出：
```
📋 Specs (18 total)

┌─────┬────────────────────────┬──────────────┬──────────┐
│ ID  │ Name                   │ Status       │ Priority │
├─────┼────────────────────────┼──────────────┼──────────┤
│ 042 │ auth-refactor          │ in-progress  │ high     │
│ 043 │ api-redesign           │ planned      │ medium   │
│ 044 │ dashboard-performance  │ complete     │ low      │
└─────┴────────────────────────┴──────────────┴──────────┘
```

### 按状态过滤

```bash
# 仅显示进行中的Spec
harnspec list --status in-progress

# 显示计划中的Spec
harnspec list --status planned

# 显示已完成的Spec
harnspec list --status complete
```

### 按标签过滤

```bash
# 显示带特定标签的Spec
harnspec list --tag api

# 显示带多个标签的Spec
harnspec list --tag api --tag backend
```

### 按优先级过滤

```bash
# 高优先级Spec
harnspec list --priority high

# 低优先级Spec
harnspec list --priority low
```

## 按内容搜索

### 基本搜索

```bash
# 搜索所有Spec内容
harnspec search "authentication"
```

搜索范围：
- Spec标题和描述
- 所有 README.md 内容
- 子 Spec文件（DESIGN.md、TESTING.md 等）
- frontmatter字段

### 使用过滤器搜索

```bash
# 在特定状态中搜索
harnspec search "API" --status in-progress

# 使用标签过滤器搜索
harnspec search "cache" --tag performance

# 组合多个过滤器
harnspec search "refactor" --status planned --tag backend --priority high
```

### 不区分大小写搜索

搜索默认不区分大小写：
```bash
harnspec search "api"      # 匹配 "API"、"api"、"Api"
harnspec search "JWT"      # 匹配 "jwt"、"JWT"、"Jwt"
```

## 依赖导航

```bash
# 显示Spec的依赖关系
harnspec deps 042
```

输出：
```
042-auth-refactor 的依赖图 (Dependency Graph)

依赖于：
  → 041-jwt-library [complete]       # 开始前需要

被需要：
  ← 043-api-redesign [planned]       # 在 042 完成前被阻塞
  ← 044-dashboard [planned]

相关Spec：
  ⟷ 045-security-audit [in-progress]  # 协调工作
```

导航关系：
- **依赖于（→）**：此Spec在开始前需要什么
- **被需要（←）**：等待此Spec的内容被阻塞
- **相关（⟷）**：信息连接

**另见**：[依赖关系](/zh-Hans/docs/guide/usage/cli/project-management#dependencies)

## 高级过滤

### 组合多个标准

```bash
# 计划中的高优先级后端工作
harnspec list --priority high --tag backend --status planned

# 进行中的 API 工作
harnspec list --status in-progress --tag api

# 已完成的安全工作
harnspec list --status complete --tag security
```

### 排除已归档

默认情况下，`harnspec list` 排除已归档的Spec：

```bash
# 默认：排除已归档
harnspec list

# 包括已归档
harnspec list --archived
```

## 常见搜索模式

### 查找要开始的工作

```bash
# 准备开始的工作是什么？
harnspec list --status planned --priority high
```

### 检查活动工作

```bash
# 进行中的是什么？
harnspec list --status in-progress

# 谁在做什么？
harnspec list --status in-progress
# （如果设置则显示负责人）
```

### 审查已完成的工作

```bash
# 最近完成的
harnspec list --status complete

# 按标签完成的工作
harnspec list --status complete --tag api
```

### 检查依赖关系

```bash
# 准备开始的是什么？
harnspec list --status planned

# 检查是否满足依赖关系
harnspec deps <spec-id>
# （验证所需Spec已完成）
```

## 导航提示

**做：**
- ✅ 使用 `harnspec search` 进行基于内容的发现
- ✅ 使用带过滤器的 `harnspec list` 进行聚焦视图
- ✅ 使用 `harnspec deps` 了解关系
- ✅ 查看[看板与统计](/zh-Hans/docs/guide/usage/cli/project-management#board-stats)获得可视化概览

**不要做：**
- ❌ 手动 grep specs/ 目录
- ❌ 依赖文件系统命名（改用 ID/搜索）
- ❌ 忘记已归档的Spec存在（使用 --archived 包括它们）

## 键盘驱动工作流程

### 快速发现流程

```bash
# 1. 项目状态如何？
harnspec list --status in-progress

# 2. 我应该做什么？
harnspec list --status planned --priority high

# 3. 这个Spec是关于什么的？
harnspec view <spec-id>

# 4. 开始工作
harnspec update <spec-id> --status in-progress
harnspec open <spec-id>
```

### 研究流程

```bash
# 1. 查找相关Spec
harnspec search "authentication"

# 2. 查看有希望的Spec
harnspec view 042

# 3. 检查相关工作
harnspec deps 042

# 4. 阅读相关Spec
harnspec view 041
```
